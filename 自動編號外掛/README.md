# 自動編號外掛 — 使用說明

> 適用版本：v1.0.0
>
> 將 [供應商編號系統設計文件](../CLAUDE.md) 的「Counter App 發號機制」封裝為 kintone 外掛：
> **於特定狀態下儲存時跳出確認訊息，儲存成功後由 Counter App 發號並回寫編號。**

---

## 功能說明

1. 使用者於符合條件的狀態下按下「儲存」。
2. 外掛跳出**確認訊息**（採用 SweetAlert2，未載入時退回瀏覽器原生 confirm）。
3. 確認後記錄存檔。
4. 記錄**儲存成功**的當下，外掛依「供應商分類」決定編號規則：
   - **發號類**（境外、NX 集團等）→ 向 Counter App 以 **revision 樂觀鎖** 遞增取號（如 `GO000001`），回寫至編號欄位。
   - **抄錄類**（國內供應商）→ 走**旁路**，直接抄錄既有欄位（如 8 碼統一編號），不經過 Counter App。
5. 完成後顯示成功訊息，並重新整理頁面以呈現編號。

> 採「儲存成功後才發號」可避免使用者取消或驗證失敗造成跳號（詳見設計文件第五節）。

---

## 設計對應

| 設計文件概念 | 外掛實作 |
| --- | --- |
| 複合鍵（app_id + category_key）定位發號機 | 執行時自動帶入 `kintone.app.getId()`，查詢使用 `source_app_id` + `category_key` |
| revision 樂觀鎖防止撞號 | `issueSerial()` 以帶 `$revision` 的 PUT 更新，遇 `GAIA_CO02` 自動重試（次數可設定） |
| 週期歸零（NONE/YEARLY/MONTHLY/DAILY） | `getPeriodTag()` 比對 `period_tag`，跨週期時歸零，動作包覆於樂觀鎖內 |
| 國內供應商走旁路抄錄統一編號 | 分類規則採「抄錄欄位」模式並搭配 `taxId8` 驗證 |
| 純前端無後端、Token 風險 | Counter Token 於前端以 fetch 帶入；安全性依賴 kintone 權限與編號欄位唯一性（詳見設計文件第八節） |

## 安裝與設定

### 1. 上傳外掛
系統管理 → 外掛程式管理 → 匯入 `plugin.zip`。

### 2. 加入 App
目標業務 App → App 設定 → 外掛程式 → 勾選本外掛 → 更新 App。

### 3. 設定畫面（齒輪圖示）

| 區塊 | 欄位 | 說明 |
| --- | --- | --- |
| **1. 計數器** | Counter App ID | 共用發號中台的 App ID |
| | Counter API Token | 建議填寫，須具「記錄編輯」權限，使用者即使無權限亦能發號 |
| | 本 App Token | 選填，供回寫編號使用；流程推進後使用者已無編輯權時需要 |
| | active 查詢條件 | 預設 `active in ("啟用")`，須對應 Counter App 的啟用值 |
| **2. 業務欄位** | 編號欄位 | 寫入目標（建議勾選「值的唯一性」） |
| | 分類欄位 | 決定套用哪條規則（如「供應商分類」） |
| **3. 觸發** | 觸發時機 | create.submit／edit.submit／process.proceed，可複選 |
| | 確認訊息文字 | 儲存前彈出的提示文字 |
| | 狀態條件 / 到狀態 / 動作名稱 | `*` 表示任意，或限定特定狀態才發號 |
| | 併發重試次數 | 預設 5 |
| **4. 分類規則** | 分類值 | 須與分類欄位選項完全相同（將直接以此值作為 Counter App 的 `category_key` 查詢） |
| | 模式 | 發號 / 抄錄 |
| | 來源欄位（抄錄用） | 抄錄模式填寫來源欄位代碼；發號模式留空 |
| | 驗證 | 抄錄模式可選「統一編號 8 碼」 |

### 設定範例（對應設計文件四筆計數器）

| 分類值 | 模式 | 參數 |
| --- | --- | --- |
| 國內供應商 | 抄錄欄位 | `統一編號`（來源欄位代碼） |
| 境外供應商 | 發號 | （無；用分類值本身） |
| NX集團供應商 | 發號 | （無；用分類值本身） |

> 一般供應商 App（685）與營運供應商 App（686）套用**同一份外掛**，`source_app_id` 自動帶入，兩者互不衝突。

### 如何調整編號樣式（RN000001 → RN-001）

**樣式不設於本外掛，亦無須修改程式**。樣式由 **Counter App 該筆發號機的 `number_format` 欄位**決定，後台維護人員直接編輯該筆記錄即可：

| 想要的號碼 | `number_format` | `pad` |
| --- | --- | --- |
| `RN000001`（預設） | 留空 或 `{prefix}{seq}` | 6 |
| `RN-001` | `{prefix}-{seq}` | 3 |
| `RN-001`（鎖 3 碼） | `{prefix}-{seq:3}` | 任意 |
| `2026-RN-0001` | `{YYYY}-{prefix}-{seq}` | 4 |

可用 token：`{prefix}`、`{seq}`、`{seq:N}`（補零 N 位）、`{period}`、`{YYYY}`/`{YY}`/`{MM}`/`{DD}`。完整說明請見[設計文件 §3.4](../CLAUDE.md)。

> 由於每個 App 使用各自的 Counter 記錄，**A App 維持 `RN000001`、B App 改為 `RN-001` 互不影響**，這正是「將樣式置於發號機本身」的優點。修改後**毋須重新打包外掛**。

## 重新打包（修改程式後）

修改 `contents/dist/*.js` 後（`desktop.js` 與 `mobile.js` 內容須一致，修改後務必同步），使用 **本外掛專屬私鑰** 重新打包：

```bash
npx @kintone/plugin-packer contents --ppk auto-numbering.ppk --out plugin.zip
```

- 私鑰 `auto-numbering.ppk`（公鑰 `auto-numbering.pub`）為本外掛專用，**請妥善保管、切勿外流**。
- 使用相同的 `.ppk` 可維持相同的 plugin ID，於後台「更新」即可覆蓋升級，且設定會自動保留。
